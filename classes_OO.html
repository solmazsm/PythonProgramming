Introducing object-oriented

A object is a physical thing that we can sense, feel, and manipulate.

An object is an instance of a class, which is a blueprint or template defining its properties (attributes) and behaviors (methods). Objects encapsulate data and related functionalities, allowing for modular and structured programming. For example, let's consider a "Car" class. An object of the "Car" class would represent a specific car instance with its unique characteristics and behaviors. The object would have attributes like "color," "model," and "speed," and methods such as "start," "accelerate," and "brake." Multiple car objects can be created based on the same class, each with its own state and behavior.
